# 强制规则（AI 行为约束，优先级仅次于流程门禁）

> 本文件规定 AI 的**行为约束**（交互、Git、宏、路径、迭代上限）。
> 流程的**顺序与门禁纪律**（必须按步骤顺序执行、不可跳过门禁、以引擎 JSON 为准）由 `flow.yaml` 的步骤表 + `SKILL.md` 铁律统一规定，二者互补、不重复。

1. **交互询问与进度展示规则** — seq 2 只确认使用当前配置或用户已修改的最新配置，必须等待回答，选择已修改后必须执行 `--reload-config`；seq 3 只基于最终 `configProjects` 逐项目选择执行模式；seq 4 只快速审查相关源码和历史，不提问；seq 5 使用 VS Code `askQuestions` 的明确选项询问复现情况、动态影响项目、异常类型和已有证据，每问允许自由补充，最后再单独询问并等待“请问还有什么补充？”。运行中仅固定阻塞场景触发 `🛑人工暂停`。当配置 `ai_progress_display=true` 时，每次调用引擎之前必须先发送独立进度消息，内容包括上一次运行的步骤/动作、简要结果以及本次调用目的；首次调用说明初始化并准备进入步骤 1。引擎返回后不重复展示 `user_display`，只保存其步骤信息供下一次调用前组织进度。不得为进度增加引擎调用。配置为 `false` 时禁止输出流程进度。进度不得扩写为分析、根因、日志或结论汇报。暂停场景细则见 `refs/pause-scenarios.md`。

2. **Git 版本管控规则** — 全程禁止 `git push`，所有分支/提交/回退仅在本地操作；临时 CHESHI 代码与正式业务修复代码强制隔离。详见 `refs/git-rules.md`。

3. **CHESHI 统一宏规范** — 全局仅使用 `CHESHI` 单一调试总开关（Bit 位掩码或数值分级）；凡调试结束需要删除的新增代码均须完整包裹，包括头文件引用、宏、类型/声明、变量、参数、辅助函数、采集器、缓冲区、Flush 及调用路径，不得只包裹打印。通信层/ISR 仅采集快照，由 `main` 主循环输出；宏定义集中写在 `main.c` 头部，调试结束整段删除并恢复临时工程配置。详见 `refs/cheshi-macro.md`。

4. **动态路径加载规则** — 流程引擎的 `--project` 为历史兼容参数名，必须传当前 VS Code **工作区根目录**，不得传 Skill 仓库目录或某个 Keil 工程目录。工作区配置、状态、日志和报告的路径必须由 `scripts/path_config.py` 根据 `scripts/skill-config.json` 生成，禁止在 Python、YAML 或 Markdown 中重复拼接；实际源码目录和工程文件从配置的 `projects[*]` 解析。日志文件必须包含项目下标和名称，禁止多项目覆盖。Keil 默认路径由集中配置自动填入，不询问用户。详见 `refs/config-format.md` 和 `refs/runtime-config.md`。

5. **8 轮迭代上限** — 自动加打印满 8 轮仍无法定位故障时，必须触发人工求助。详见 `refs/debug-loop.md`。

6. **持续运行规则（运行中不可随意停下）** — 流程一旦进入执行，AI 必须持续驱动引擎按 `seq` 推进，**非必要情况一律不得停下/退出流程**：
   - **非必要情况（禁止停下）**：需要修改代码、需要向用户询问、需要等待确认、某一步暂未定位等——这些本就是流程内的正常环节（`edit_source` / `ask_user` / 迭代回退），必须在线内完成，不得借机中止整个流程或要求用户手动接管。
   - **必要情况（允许 `🛑人工暂停`）**：仅限真正阻断流程、AI 无力绕过的硬阻塞，例如：
     1. 程序卡死、目标板死机/看门狗失效，必须断电重启才能继续；
     2. 硬件缺失或不可用（无目标板、无下载器、串口/调试器丢失、供电异常），流程客观上无法推进。
  - 除以上必要情况外，AI 不得主动停下流程；即便需要用户补充信息，也应走流程内的 `ask_user` 步骤（如 seq 13 追问环节），而非终止运行。必要暂停场景细则见 `refs/pause-scenarios.md`。

7. **编译下载跳过参数由 AI 决策** — 禁止询问用户是否设置 `skipBuild` / `skipFlash`。用户或 AI 希望开始时先观察板上现有固件串口时，自动跳过编译下载并启用 `observeExistingSerial`；用户明确要求结束流程时自动启用 `finishRequested`，编译失败只能作为报告中的未验证项，不得阻塞 CHESHI 清理、报告生成和流程完成。
