# 工作流总览

```
Step 1: 发现问题
          │
          ▼
Step 2: config_reader.py 读取配置
          │
          ▼
Step 3: 分析日志 / 定位可疑代码
          │
          ▼
Step 4: 需人工操作提速?
         ├── 是 🛑 → 三类暂停场景之一 → 用户完成操作并反馈 → 回到 Step 3
         └── 否 → 添加 CHESHI 调试打印
                    │
                    ▼
Step 5: keil_build.py 编译
         ├── 报错 → 回到 Step 5 修正
         └── 成功 → keil_flash.py 下载固件
                      ├── 下载失败 → 是否重试?
                      │              ├── 是 → 重新下载
                      │              └── 否 🛑 → 三类暂停场景之一
                      └── 成功 → serial_read.py / serial_monitor.py 抓日志
                                   │
                                   ▼
Step 6: 信息足以定位根因?
         ├── 不足，且迭代≤8 → 回到 Step 5 (添加更多打印)
         ├── 不足，且迭代>8 → 🛑 三类暂停场景之一 (求助)
         └── 充足 → 修改业务代码修复
                      │
                      ▼
Step 7: 编译下载验证
          │
          ▼
Step 8: 故障解决?
         ├── 否 → 回到 Step 3
         └── 是 → 清理 CHESHI 宏
                    │
                    ▼
Step 9: 全量回归验证
          │
          ▼
Step 10: 输出报告
```
